Sealed File Inspector

Drop a .sealed file to verify the issuer signature and inspect the manifest. No upload.

Inspection runs entirely in your browser. The file is parsed locally, the Ed25519 signature is verified via the WebCrypto API, and nothing leaves your device unless you explicitly click a registry lookup button below. A modern browser is required: Safari 17 or later, Chrome 113 or later, or Firefox 130 or later.
Drop a .sealed file here
or click to choose one from your device

Don't have a sealed file? Try one of these samples: hello-alice.sealed (valid signature, not decryptable), hello-alice-tampered.sealed (signature invalid), tutorial.sealed (classic suite, decryptable with the tutorial identity), tutorial-hybrid.sealed (hybrid post-quantum suite, decryptable with the hybrid tutorial identity). Drop any of them into the inspector above.

Decrypt in browser

Paste or upload the recipient identity.json (as produced by oversight keygen). Private keys stay in memory; decryption runs locally using WebCrypto for X25519 + HKDF, a vendored copy of @noble/ciphers for XChaCha20-Poly1305, and a vendored copy of @noble/post-quantum for ML-KEM-768 when the file is hybrid (post-quantum). Both classic and hybrid suites decrypt here.

Registry lookup

The manifest declares a registry at (none). Clicking a button below will send a request to that URL. This discloses to the registry operator that someone is inspecting this file_id.
